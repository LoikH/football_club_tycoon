## ⚡ 2. Core Architectural Constraints

To keep the game fully portable and lightweight, the codebase rigorously adheres to the following constraints:
1.  **CORS & File Protocol Portability:** Absolutely **no ES6 Modules (`import`/`export`)**. All scripts are loaded sequentially in `index.html` via global vanilla JS tags. This ensures the game runs seamlessly when launched directly from the local filesystem (`file://`) or packaged as an offline web build (e.g., inside an Itch.io zip sandbox).
2.  **No Native Popups:** The standard browser `alert()` and `confirm()` functions are strictly banned. All notifications, game over screens, and validation states are rendered through custom in-game modal layers (`.modal-overlay`) to preserve visual immersion and responsive rendering.
3.  **Lightweight Synthetic Audio:** No heavy `.mp3` or `.wav` sound files are loaded. Sound effects are synthesized on-the-fly using the browser's native **Web Audio API** via customizable oscillator waves (`sine`, `triangle`, `sawtooth`).
4.  **Full Responsiveness & Mobile Friendly:** Layouts are engineered using responsive flexbox/grid units, ensuring smooth adaptations on screen sizes down to `320px` width. Pointer events handle hover tilts gracefully on desktops while disabling them on mobile touch screens.

## 🔄 3. State Machine & Math Formulas (`js/gameState.js`)

The global game state is managed in the `GameState` class.

### Key Variables:
*   `money` (Number): The current disposable club treasury.
*   `totalEarned` (Number): Lifetime accumulated dollars (used to compute the Division/League title).
*   `prestigeCups` (Number): Number of Rebirth seasons completed (each cup gives a permanent +100% multiplier).
*   `upgrades` (Object): Map of upgrade IDs to their current purchased level (`{ [upgradeId]: level }`).
*   `cards` (Object): Map of player IDs to their current level (`{ [cardId]: level }`), capped at **level 5**.
*   `lang` (String): Active language setting (`"fr"` or `"en"`), persisting through saves.

### Math Formulation:
1.  **Upgrade Purchase Cost:**
    The cost to buy the next level of an infrastructure upgrade is exponential:
    $$\text{Cost} = \text{baseCost} \times (\text{costMultiplier})^{\text{currentLevel}}$$
2.  **Upgrade Base Yield:**
    The raw passive yield generated by an upgrade is linear:
    $$\text{RawYield} = \text{level} \times \text{baseYield}$$
3.  **Holographic Cards Multiplier:**
    Each owned player card contributes a passive boost based on its level:
    $$\text{GlobalCardsMultiplier} = \sum_{\text{owned}} (\text{baseBoost} \times \text{level})$$
4.  **Global Passive Income per Second:**
    Upgrade yields are combined, scaled by the card collection multiplier, multiplied by the Prestige cups bonus, and boosted by active mini-game buffs:
    $$\text{IncomePerSec} = \left( \sum \text{RawYield} \right) \times (1 + \text{GlobalCardsMultiplier}) \times (1 + \text{prestigeCups}) \times \text{passiveMultBonus}$$
5.  **Active Click Value:**
    $$\text{ClickValue} = \left( 1 + \left( \sum \text{RawYield} \right) \times 0.08 \right) \times (1 + \text{GlobalCardsMultiplier}) \times (1 + \text{prestigeCups}) \times \text{clickMultBonus}$$

## 🏆 5. Prestige Rebirth Season

When the user reaches **`$1,000,000,000 ($1.00B)`** in active treasury, the Rebirth button (`#btn-prestige`) starts pulsating with a glowing neon gold animation.
*   **Validation Screen:** Toggles a beautiful, descriptive custom modal overlay (`#rebirth-confirm-modal`) outlining the benefits and consequences.
*   **Reset Consequences:** `money` resets to `0`, all purchased infrastructure levels in `upgrades` reset to `0`.
*   **Preserved Elements:** The entire roster of holographic player cards and their gathered levels (capped at `5`) are fully preserved.
*   **Prestige Reward:** Increments `prestigeCups` by `1`, applying a permanent, lifetime cumulative **+100% multiplier** to all click and passive revenue.

## 🎮 6. Interactive Active Mini-Games (`js/main.js`)

Every second, if the standard roster is not yet complete (less than 24/24), there is a random chance to spawn a floating balloon widget (`#random-event-btn`) carrying one of three active events.

### Mini-game 1: Penalty Shootout (`#penalty-modal`)
A target indicator bar oscillates back and forth across a timing gauge at a rate of `2.4%` width per frame.
*   **Winning Conditions:** The user must click `#btn-shoot-penalty` when the cursor lies exactly inside the expanded green target zone **between 35% and 65% width** (covering 30% of the bar).
*   **Success Reward:** Plays a triad chime. Grants immediate cash equivalent to **5 minutes of passive income** (`state.incomePerSec * 300`). Toggles victory modal.
*   **Failure Consequence:** Plays a sawtooth buzz. Inflicts a public disappointment debuff: **-20% passive income for 60 seconds** (`passiveMultBonus = 0.8`). Toggles disappointment modal.

### Mini-game 2: Press Conference (`#press-modal`)
Prompts the user with random, risky journalistic inquiries (astronomical transfers, referee bribery) with two distinct choices (Diplomatic `Option A` or Arrogant/Provocative `Option B`).
*   **Option A (Diplomatic):** Success rate: `75%`. Success gives `+15%` passive income for 120s. Failure gives `-5%` click value for 60s.
*   **Option B (Arrogant):** Success rate: `35%`. Success triggers a viral meme boosting Hype: `+50%` passive income for 120s. Failure triggers massive supporter outcry: `-30%` passive income for 90s.
*   All outcome descriptions are loaded dynamically in the active language.

### Mini-game 3: Financial Fair Play Audit (`#audit-modal`)
Forces the user to quickly sign off on accounts before UEFA inspectors seize club funds.
*   **Action:** The user clicks "Start Falsification" which launches a **9.0-second countdown timer**.
*   **Gameplay:** Bouncing paper targets (`📄`) spawn randomly on screen. The user must click **5 documents** before the timer hits 0.
*   **Success Reward:** Grants UEFA audit clearance: **+30% passive income for 3 minutes** (`passiveMultBonus = 1.30`).
*   **Failure Consequence:** UEFA uncovers bribery, applying a heavy immediate accounting fine: **seizes -15% of total active club treasury**.

### 🤙 7. The Legend of Ronaldinho: Secret Quest Keepy-Uppy (`#jongle-ball`)
The ultimate end-game active event.
1.  **Condition:** The user must have completed the standard collection (unlocked all 24 cards, `24/24` standard deck progress).
2.  **Trigger:** Every second, if Ronaldinho is locked, there is a **1.5% chance** to automatically spawn a golden soccer ball (`#jongle-ball`).
3.  **Introduction Modal:** Pauses physics, pops up `#outcome-modal` displaying instructions, and waits for user click to close before beginning.
4.  **Keepy-Uppy Physics:** The ball falls from `y=20` (inside viewport) with low gravity (`0.15` velocity increment per frame) and minor side drift, making the first click relaxed and reactive.
5.  **Interactivity:** Clicking the ball propels it upward (`ballVelY = -6.8`) and applies a random horizontal deflection. Plays pitch-shifting triangle chimes and spawns gold-glowing floaties (`JONGLE +X`).
6.  **Victory:** The user must keep the ball in the air for **3 consecutive clicks**. Upon 3 hits, calls `state.addCard("ronaldinho")` and opens the giant reveal modal. If the ball touches the bottom of the screen, the event fades, prompting the player to wait for the next spawn.

## 🌐 8. Dynamic Localization & Translations (`js/translations.js`)

Full French/English bilingual support is integrated on-the-fly.
*   **Dynamic Language Toggle (`#btn-lang-toggle`):** Instantly flips `state.lang` between `"fr"` and `"en"`, calls `ui.translateUI(state)` to overwrite all DOM text nodes, rerenders upgrades/deck dynamically, and saves preference.
*   **Card details:** Handled cleanly since `CARDS_DATA` stores biographical desc objects like `{ fr: { desc: "..." }, en: { desc: "..." } }`.
*   **League Division Progression:** Automatically translates and computes league titles on HUD depending on active save state language and lifetime earned money.

## 🔊 9. Synthetic Audio Synth Engine (`js/main.js`)

Synthesizes sound effects via the browser's Web Audio API `AudioContext`.
*   **Instant Mute Control:** Toggling the `#btn-sound-toggle` button switches `audio.isMuted`. Setting this to `true` instantly short-circuits the first line of the `playNote` function, guaranteeing complete, global silence in all game modules.
*   **HUD Button feedback:** Swaps between speaker icons (`🔊 SON` / `🔇 SON`) and changes button border neon glows dynamically from cyan (active) to red (muted).
